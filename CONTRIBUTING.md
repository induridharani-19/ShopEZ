# Contributing to ShopEZ

We welcome contributions to ShopEZ! To contribute, please follow these guidelines:

## Code Style
- Use ESM imports (`import / export`) in Javascript backend and frontend files.
- Keep CSS styles modular.
- Document any new REST API endpoints with inputs and outputs in `docs/user_manual.md`.

## Commit Message Guidelines
Use clear descriptive commit prefixes:
- `feat:` for new features (e.g., `feat: added AI recommendation shelf`)
- `fix:` for bug fixes (e.g., `fix: secured user query admin route`)
- `docs:` for documentation updates
- `test:` for adding or modifying test suites

## Workflow
1. Fork the repository and create a branch: `git checkout -b feature/your-feature-name`.
2. Commit your modifications with test verifications.
3. Push to your branch and open a Pull Request.
